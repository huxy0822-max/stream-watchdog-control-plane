# 超级管理员使用教程

这份文档面向你自己使用。当前定位不是开放式 SaaS 平台，而是：

- 你自己维护一个平台管理员后台
- 给客户发 `CDK`
- 客户进入普通用户后台使用直播监控和恢复功能

后续如果要替别人单独部署一套，再在这套基础上扩展。

## 1. 你现在负责什么

你在超级管理员后台主要做这几件事：

- 创建和维护客户空间
- 给客户创建账号，或者生成 `CDK`
- 维护平台运行参数
- 管理邮件通知
- 检查全局异常
- 帮客户录入服务器和直播流，或者让客户自己录入

## 2. 超级管理员后台和普通用户后台的区别

超级管理员能看到：

- 全部客户空间
- 全部服务器
- 全部直播流
- CDK 列表
- 平台设置
- 邮件设置

普通用户只能看到：

- 自己空间里的服务器
- 自己空间里的直播流
- 自己空间里的异常
- 自己空间里的分组视图

## 3. 建议你的开通流程

### 方案 A：你手工建客户空间

适合大客户、定制客户。

流程：

1. 先建客户空间
2. 再建客户管理员账号
3. 帮对方录入服务器和直播流

### 方案 B：你生成 CDK

适合标准化销售。

流程：

1. 在后台生成 CDK
2. 发给客户
3. 客户自己用 CDK 开通
4. 客户自己登录普通用户后台配置服务器和直播流

你当前主要应该用这个方案。

## 4. 如何创建客户空间

在“客户空间与开通管理”里填写：

- `客户空间名称`
- `客户空间标识`
- `状态`
- `到期时间`
- `最大账号数`
- `最大服务器数`
- `最大直播流数`
- `备注`

建议：

- `名称` 用对外显示的名字
- `标识` 用系统内部稳定值，不要随意改
- 配额尽量跟你的实际卖法保持一致

## 5. 如何生成 CDK

CDK 表单里主要填：

- `CDK`
  可留空自动生成，也可以自己手填
- `套餐名称`
- `有效天数`
- `最大账号数`
- `最大服务器数`
- `最大直播流数`
- `备注`

建议你按套餐做模板，例如：

- `VIP-30D-10S-100L`
- `VIP-90D-20S-300L`

## 6. 如何帮客户规划分组

客户后台的巡检视图是：

- 分组
- 服务器
- 直播流

所以你在给客户培训时，最好直接告诉他们按这几种思路分组：

- 地区
- 项目
- 业务线
- 主备

不要让客户把几十台服务器全扔在一个分组里。

## 7. 平台设置怎么理解

### 巡检间隔

越小越敏感，但对系统压力越高。

建议：

- 常规先用 `20` 秒

### SSH 超时

建议：

- 常规 `10` 到 `15` 秒

### 默认验证延迟

重推后给 `ffmpeg` 一点时间起来。

建议：

- 常规 `5` 到 `10` 秒

### 会话时长

决定后台登录多久过期。

### 事件保留数

决定后台最多保留多少事件记录。

## 8. 邮件设置

如果你要启用邮件通知，需要准备：

- SMTP Host
- SMTP Port
- 是否 TLS
- SMTP 用户名
- SMTP 密码
- 发件邮箱
- 收件邮箱

如果暂时没有发件邮箱，可以先不填，系统监控和自动恢复仍然能工作。

## 9. 你每天应该怎么用

推荐流程：

1. 先看“异常优先处理”
2. 再看全局直播矩阵
3. 再按分组巡检视图抽查
4. 最后看事件记录

不要从事件流开始看，因为事件流只是补充信息，不是主视图。

## 10. 给客户部署独立一套时要注意什么

后续如果你要卖“单独部署版”，要重点检查：

- 目标服务器能否 24 小时运行
- 目标服务器能否对外 SSH 到客户推流机
- Docker 是否可用
- 端口和现有网站是否冲突
- 反向代理是否可配

部署前至少验证：

- 后台能启动
- 普通用户能登录
- 能保存服务器
- 能保存直播流
- 能巡检
- 能手动恢复

## 11. 当前版本边界

当前版本已经适合：

- 你自己作为唯一平台管理员
- 用 CDK 给客户开普通使用权限
- 卖“服务”而不是开放式 SaaS

当前版本暂未完成：

- 统一 HTTPS 自动化
- 直接支付
- 小程序端
- YouTube API 二次确认
- 黑屏识别

所以你对外卖的时候，应该把定位说清楚：

- 这是“直播监控与自动恢复服务后台”
- 不是“全自动 AI 直播 SaaS 平台”
