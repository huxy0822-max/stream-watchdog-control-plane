# 超级管理员使用教程

这份文档面向平台所有者，也就是你自己。当前主要定位是“你维护一个控制台，为客户开通和运维直播监控服务”。

## 1. 登录入口

- 超级管理员登录：`/admin/login`
- 普通用户登录：`/customer/login`
- CDK 开通：`/redeem`

现在入口已经拆开：

- 主域名 `/` 不再展示超级管理员入口
- 超级管理员只能通过 `/admin/login` 进入

## 2. 你能管理什么

超级管理员后台可以管理：

- 全部客户空间
- 全部客户账号
- 全部分组
- 全部服务器
- 全部直播流
- CDK
- 平台运行参数
- 邮件通知
- 管理服务器资源和程序自身资源

## 3. CRM 化后台结构

现在后台已经改成左侧导航 + 单模块页面结构。

主要模块包括：

- 总览
- 直播矩阵
- 分组巡检
- 服务器
- 直播流
- 客户空间
- 客户账号
- CDK 管理
- 资源监控
- 平台设置
- 通知设置
- 事件记录
- 账号安全

## 4. 客户开通方式

### 方式 A：你手动创建客户空间

适合高客单、定制客户：

1. 创建客户空间
2. 创建客户管理员账号
3. 帮客户录入分组、服务器和直播流

### 方式 B：你发 CDK

适合标准化服务：

1. 生成 CDK
2. 发给客户
3. 客户自己去 `/redeem` 开通
4. 客户自己去 `/customer/login` 使用

## 5. 客户空间和配额

创建客户空间时重点填：

- 名称
- slug
- 状态
- 到期时间
- 最大账号数
- 最大服务器数
- 最大直播流数
- 备注

建议 slug 稳定，不要频繁改。

## 6. CDK 管理

CDK 页面现在支持一次生成多个。

常用字段：

- 自定义 CDK
- 生成数量
- 套餐名称
- 有效天数
- 最大账号数
- 最大服务器数
- 最大直播流数
- 备注

建议按套餐模板命名，例如：

- `VIP-30D-10S-100L`
- `VIP-90D-20S-300L`

如果是标准化服务，直接一次生成一批同套餐 CDK 更高效。

## 7. 服务器接入新流程

现在不一定要手工录全部直播流。

推荐流程：

1. 先保存服务器 SSH 信息
2. 点击“SSH 识别当前推流”
3. 系统自动扫描当前正在运行的 `ffmpeg`
4. 自动识别媒体文件路径和 YouTube 推流码
5. 自动导入直播流配置

这一步特别适合接入已经在播的老服务器。

## 8. 给客户培训时重点说什么

客户新建直播流时，不需要自己写整条 `ffmpeg` 命令。

推荐他们这样做：

1. 选择服务器
2. 填写媒体文件名或完整路径
3. 填写 YouTube 推流码
4. 点击“保存并立即开播”

这样能明显减少误操作。

## 9. 日常运维建议

每天优先看：

1. 总览
2. 资源监控
3. 直播矩阵
4. 分组巡检
5. 事件记录

## 10. 仓库自动化

仓库里现在已经带上：

- Node.js CI
- GitHub Packages 发布工作流
- Datadog Synthetics 工作流

Datadog 启用前需要配置：

- `DD_API_KEY`
- `DD_APP_KEY`
- `DD_SYNTHETICS_PUBLIC_IDS`
- 可选 `DATADOG_SITE`

如果这些值没配，Datadog 工作流会自动跳过，不会把仓库状态打红。
