# 超级管理员使用教程

这份文档面向平台所有者，也就是你自己。当前主要定位是“你维护一个控制台，为客户开通和运维直播监控服务”。

## 1. 登录入口

- 超级管理员登录：`/admin/login`
- 普通用户登录：`/customer/login`
- CDK 开通：`/redeem`

现在入口已经拆开，不再是同页直接混合登录。

## 2. 你能管理什么

超级管理员可以管理：

- 全部客户空间
- 全部分组
- 全部服务器
- 全部直播流
- CDK
- 平台运行参数
- 邮件通知
- 管理服务器资源和本程序资源占用

普通用户只能看到自己的空间数据。

## 3. 推荐开通流程

### 方案 A：你手动创建客户空间

适合高客单、定制客户：

1. 创建客户空间
2. 创建客户管理员账号
3. 帮客户录入分组、服务器和直播流

### 方案 B：你发 CDK

适合标准化服务：

1. 生成 CDK
2. 发给客户
3. 客户自己去 `/redeem` 开通
4. 客户自己去 `/customer/login` 使用

## 4. 给客户培训时重点说什么

客户现在新建直播流时，不需要自己写整条 `ffmpeg` 命令。

推荐他们这样做：

1. 选择服务器
2. 填写媒体文件名或完整路径
3. 填写 YouTube 推流码
4. 点击“保存并立即开播”

这样能明显减少误操作。

## 5. 客户空间和配额

创建客户空间时重点填：

- 名称
- slug
- 状态
- 到期时间
- 最大账号数
- 最大服务器数
- 最大直播流数
- 备注

建议 slug 稳定，不要频繁改。

## 6. CDK 建议

CDK 常用字段：

- code
- 套餐名称
- 有效天数
- 最大账号数
- 最大服务器数
- 最大直播流数
- 备注

建议按套餐模板命名，例如：

- `VIP-30D-10S-100L`
- `VIP-90D-20S-300L`

## 7. 日常运维建议

每天优先看：

1. 异常优先处理
2. 管理服务器资源
3. 直播矩阵
4. 分组巡检视图
5. 事件记录

## 8. 仓库自动化

仓库里现在已经带上：

- Node.js CI
- GitHub Packages 发布工作流
- Datadog Synthetics 工作流

Datadog 启用前需要配置：

- `DD_API_KEY`
- `DD_APP_KEY`
- `DD_SYNTHETICS_PUBLIC_IDS`
- 可选 `DATADOG_SITE`

如果这些值没配，Datadog 工作流会自动跳过，不会把仓库状态打红。

## 9. 当前这版新增点

- 登录入口拆分
- `/api/setup/status` 不再对公网匿名直接暴露
- 直播流支持显示和编辑推流码
- 支持通过“文件名/路径 + 推流码”自动生成重推命令
- 支持“保存并立即开播”
