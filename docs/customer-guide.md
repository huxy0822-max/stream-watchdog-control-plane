# 普通用户使用教程

这份文档面向普通用户或 VIP 用户。你可以通过 `CDK` 开通，也可以自己注册。登录后只会看到自己空间内的服务器、直播流和告警，不会看到别人的数据。

## 1. 登录入口

- 普通用户登录：`/customer/login`
- 普通用户注册：`/customer/register`
- CDK 开通：`/redeem`
- 超级管理员登录：`/admin/login`

主域名 `/` 现在会默认跳转到普通用户登录页；如果你已经登录过且会话还有效，会直接进入自己的后台首页。

## 2. 开通方式

### 方式 A：自己注册

1. 打开 `/customer/register`
2. 填写客户空间名称
3. 可选填写空间标识 slug
4. 填写登录账号和密码
5. 提交后系统会自动创建空间并直接登录

### 方式 B：使用 CDK 开通

1. 打开 `/redeem`
2. 填写 CDK、空间名称、账号和密码
3. 提交后系统会创建空间
4. 然后从 `/customer/login` 登录

## 3. 需要准备什么

你只需要准备：

- 推流服务器的 SSH 信息
- 服务器分组方案
- 每一路直播对应的媒体文件
- 每一路直播对应的 YouTube 推流码

当前版本不需要你自己申请：

- YouTube API Key
- Google OAuth
- YouTube Live Streaming API 授权

## 3. 先规划分组

后台是按下面这个层级看的：

- 分组
- 服务器
- 直播流

建议按地区、项目、业务线或主备关系分组，不要把几十台服务器全堆在一个分组里。

## 4. 添加服务器

在“服务器管理”里填写：

- 所属分组
- 服务器名称
- 主机地址
- SSH 端口
- SSH 用户名
- SSH 密码
- 备注

## 5. SSH 自动识别当前推流

如果服务器上本来就已经在直播，不需要你手工一条条重录。

操作流程：

1. 先保存服务器
2. 点击“SSH 识别当前推流”
3. 系统会登录服务器，扫描正在运行的 `ffmpeg`
4. 自动识别媒体文件路径和 YouTube 推流码
5. 自动导入到“直播流管理”

这一步非常适合接入已经在播的老服务器。

## 6. 新建直播流

推荐直接使用简化模式。你只需要填写：

- 所属服务器
- 媒体文件名或完整路径
- YouTube 推流码

可选填写：

- 直播流名称
- 重启日志路径
- 冷却时间
- 重启窗口
- 最大重启次数
- 验证延迟

如果直播流名称留空，系统会默认使用文件名。

如果你只填 `lbo2.mp4`，系统会自动按 `/root/lbo2.mp4` 处理。

## 7. 一键开播

新建直播流时，最快的流程是：

1. 选择服务器
2. 填写文件名或完整路径
3. 填写推流码
4. 点击“保存并立即开播”

系统会自动：

- 生成 `ffmpeg` 重推命令
- 生成匹配关键字
- 保存直播流配置
- 立即尝试启动推流

## 8. 匹配关键字怎么理解

系统会用匹配关键字去识别服务器上的 `ffmpeg` 进程。

简化模式下，系统会自动把这些信息作为匹配关键字的一部分：

- 文件路径
- 推流码

不要只写这种过泛的词：

```text
ffmpeg
youtube
rtmp
```

## 9. 后台怎么看

建议按这个顺序看：

1. 总览
2. 直播矩阵
3. 分组巡检
4. 直播流管理
5. 事件记录

事件流只适合看补充信息，不适合当主界面。

## 10. 常见问题

### 服务器显示离线

检查：

- SSH IP
- SSH 端口
- SSH 用户名和密码
- 防火墙

### 直播流识别不到

检查：

- 推流码是否填写
- 文件路径是否填错
- 服务器上是否真的有对应 `ffmpeg` 进程

### 自动恢复失败

检查：

- 推流码和文件路径是否对应
- 是否已经保存成功
- 冷却时间和重启窗口是否设置得过严
