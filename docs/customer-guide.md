# 普通用户使用教程

这份文档面向通过 `CDK` 开通后的普通用户或 VIP 用户。你登录后只会看到自己客户空间内的数据，看不到别人的服务器和直播流。

## 1. 先知道这版需要准备什么

当前版本主要靠 `SSH + ffmpeg 进程检测 + 自动重推` 工作。

你需要准备：

- 你自己的服务器 IP
- 服务器 SSH 用户名
- 服务器 SSH 密码
- 服务器分组名称
- 每一路直播流对应的 `ffmpeg` 关键字
- 如果你想指定自己的重推命令，也可以准备单独的重推命令

当前版本 **不需要** 你自己去申请：

- YouTube API Key
- Google OAuth Client
- YouTube Live Streaming API 授权

如果后续平台启用 YouTube API 二次确认，那会再单独通知。

## 2. 登录后台

登录地址由管理员提供。

登录时请使用：

- 普通用户 / VIP 用户入口
- 你的登录账号
- 你的登录密码

如果你是用 `CDK` 开通的，第一次先在开通页面填写：

- 兑换码
- 客户空间名称
- 客户空间标识
- 你的管理员账号
- 你的管理员密码

开通完成后，再从普通用户入口登录。

## 3. 先设置服务器分组

系统支持按下面这种层级查看：

- 分组
- 服务器
- 服务器下面正在播的直播流

建议你按业务来分组，比如：

- `美国区`
- `欧洲区`
- `项目A`
- `项目B`
- `主力机器`
- `备用机器`

这样你后面看后台时会很清楚。

## 4. 添加服务器

进入后台后，找到“服务器管理”。

你需要填写：

- `所属客户空间`
  普通用户通常不用选，系统会自动归到你自己的空间。
- `服务器分组`
  例如 `美国区-A组`
- `服务器名称`
  例如 `US-01`
- `主机地址`
  服务器公网 IP
- `SSH 端口`
  默认一般是 `22`
- `SSH 用户名`
  例如 `root`
- `SSH 密码`
- `备注`
  可选

建议：

- 名称不要乱写，最好能让你一眼知道是哪台机器
- 分组尽量稳定，不要今天叫 `A` 明天叫 `分组1`

## 5. 添加直播流

进入“直播流管理”，一台服务器可以配置多路直播流。

你需要填写：

- `所属服务器`
- `直播流名称`
  例如 `美区女装直播间01`
- `匹配关键字`
  这是最重要的字段，用来让系统从 `ffmpeg` 进程里识别出是哪一路流
- `重启命令`
  可选，不填时系统会优先使用它学到的 `ffmpeg` 命令自动恢复
- `重启日志路径`
- `冷却时间`
- `重启窗口`
- `窗口内最大重启数`
- `验证延迟`

## 6. 匹配关键字怎么写

匹配关键字是一行一个，系统会用这些词去匹配远程服务器上的 `ffmpeg` 命令。

最推荐写法：

- 输入文件路径
- 推流目标地址里的唯一部分
- 你自己封装脚本里的唯一标记

例如：

```text
/root/1.mp4
```

或：

```text
live2/abcd-efgh-ijkl
```

不要写太泛的词，比如：

```text
ffmpeg
youtube
rtmp
```

这种会匹配到很多流，容易误判。

## 7. 重启命令要不要自己填

有两种方式：

### 方式 A：不填

如果某一路流正在正常推，系统会先学会它当前的 `ffmpeg` 启动命令。以后掉线时就按学到的命令重推。

适合：

- 你当前服务器上已经在正常推流
- 你的命令比较标准

### 方式 B：自己填

你也可以手工填一个更稳定的重推命令，比如：

```bash
systemctl restart stream-room-001
```

或者：

```bash
bash /root/restart-room-001.sh
```

更推荐这种方式，因为后续可维护性更高。

## 8. 后台怎么看

你登录后，重点看这几个区域：

### 异常优先处理

这里会单独列出：

- 异常服务器
- 异常直播流

如果这里有内容，优先处理这里。

### 直播矩阵

这里是按卡片快速看每一路直播流状态的。

### 分组巡检视图

这里会按层级展示：

- 分组
- 每个分组下面有哪些服务器
- 每个服务器下面有哪些直播流

适合做日常检查。

## 9. 常见问题

### 1) 服务器显示离线

通常检查：

- SSH IP 是否写错
- SSH 端口是否写错
- SSH 用户名密码是否写错
- 服务器防火墙是否拦截

### 2) 直播流一直识别不到

通常检查：

- 匹配关键字是不是太泛
- 关键字是不是根本没出现在 `ffmpeg` 命令里
- 当前服务器上是不是根本没有这一路流

### 3) 自动重推没有成功

通常检查：

- 是否配置了可用的重启命令
- 是否曾经让系统学到过这一路流的真实 `ffmpeg` 命令
- 重启窗口和冷却时间是不是设得太严

## 10. 推荐使用习惯

- 先把服务器分组整理清楚
- 一台服务器的每一路直播流都命名清楚
- 匹配关键字尽量用唯一值
- 能用 `systemctl restart xxx` 的尽量不要直接写长 `ffmpeg` 命令
- 先看“异常优先处理”，再看分组视图
